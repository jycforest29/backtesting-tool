1. 개요

무엇을 만들었는가

주식 투자 전략을 사전에 백테스트하고, 그 결과를 바탕으로 실제 자동매매까지 연결할 수 있는 개인용 Web 툴입니다. 모든 시세·주문은 한국투자증권 OpenAPI를 통해 단일 소스로 처리하므로, 과거 데이터와 실거래 주문이 같은 계좌·같은 데이터 소스를 공유합니다.

핵심 기능 6가지

탭역할
포트폴리오 백테스트다중 자산 비중·리밸런싱·적립식(DCA)·세금/수수료 반영 후 과거 수익률 시뮬레이션. 벤치마크(SPY, 예금 3.5% 등) 비교
단일 종목 백테스트"X년 Y월에 샀다면 지금 얼마"식 간단 수익률 계산
투자자별 매매동향개인/외국인/기관 일별 순매수 + 주가 괴리 신호 (매집·분산 감지)
자동매매 (Lv1)실시간 감시종목 가격, 지정가/시장가 수동 주문, 잔고·미체결 조회, 예상 수수료·세금 프리뷰
위기 시나리오COVID/GFC/금리인상 등 6개 프리셋 + 커스텀. 자산군별 β 기반 손실 추정
감사 로그모든 API 호출 기록 + 필터링/통계. 실계좌 주문 추적 용도

지원 시장

한국(KOSPI/KOSDAQ) · 미국(NAS/NYSE/AMEX) · 일본(TSE). 2000년대 이후 일봉 기준. 파생·코인·환율은 범위 밖.

왜 KIS 단일화? 기존에 Yahoo Finance로 시작했으나 ① 2024년 이후 401 Unauthorized 빈발 ② 비공식 API라 ToS 회색지대 ③ 시세만 제공되어 실거래로 연결 불가. KIS로 통합하면 백테스트·시세·주문·잔고가 한 계좌 안에서 일관되게 돌아감.

3. 한국투자증권 OpenAPI 활용 상세

3.1 인증 (OAuth2 + 24시간 토큰)

KisAuthService가 /oauth2/tokenP로 access_token 발급·캐싱. 발급 한도가 빡빡(일일 약 10회, 초당 1건)하므로 파일 영속화(~/.kis-token.json)로 서버 재시작 후에도 유효시간 내면 재발급 없이 재사용.

{ "token": "...", "expiresAt": 1745120334304,
  "paper": false, "appKeyHash": "a3f2..." }
주의: 실전 키와 모의투자 키는 서로 다른 앱 등록 필요. paper_trading 플래그가 잘못 설정되면 "해당 환경 TR_ID 없음" 에러로 모든 호출 실패.

3.2 시세·과거 데이터

용도엔드포인트TR_ID
국내 현재가/domestic-stock/v1/quotations/inquire-priceFHKST01010100
국내 일봉 (백테스트)/domestic-stock/v1/quotations/inquire-daily-itemchartpriceFHKST03010100
해외 현재가/overseas-price/v1/quotations/priceHHDFS00000300
해외 일봉/overseas-price/v1/quotations/dailypriceHHDFS76240000

한 번 호출에 국내는 최대 100일, 해외는 100일 응답이라 페이지네이션으로 과거로 소급. KisMarketDataService.getChartData()가 커서를 뒤로 밀면서 수집.

3.3 투자자별 매매동향

FHKST01010900으로 종목별 일별 개인/외국인/기관 순매수 수량·금액 조회. 매집/분산 신호 해석 로직:

  • BULLISH 외국인+기관 순매수 > 0인데 주가 횡보/하락 → 매집 의심
  • BEARISH 외국인+기관 순매도인데 주가 상승 → 분산(분할매도) 의심
  • NEUTRAL 수급과 주가 방향 일치

3.4 자동매매 (주문·잔고)

거래소별로 TR_ID가 다르며 실전/모의 접두 다른 점이 함정 포인트:

시장·방향실전 TR_ID모의 TR_ID
국내 매수 / 매도TTTC0012U / TTTC0011UVTTC0012U / VTTC0011U
미국 매수 / 매도TTTT1002U / TTTT1006UVTTT1002U / VTTT1001U
일본 매수 / 매도TTTS0308U / TTTS0307UVTTS0308U / VTTS0307U
국내 잔고TTTC8434RVTTC8434R
해외 잔고TTTS3012RVTTS3012R
국내 미체결TTTC0084R모의 미지원

POST 주문 시 /uapi/hashkey로 body 해시 선발급 후 hashkey 헤더에 첨부. KisAuthService.hashkey()가 담당.

3.5 Rate Limiter (필수 인프라)

KIS는 초당 호출을 엄격히 제한:

  • 실전: 18 req/sec (계좌 단위), 신규 가입 후 3일간은 3 req/sec
  • 모의: 1 req/sec
  • 토큰 발급: 1 req/sec (별도 버킷)
  • 초과 시 EGW00201 응답 → HTTP 500으로 내려옴

KisRateLimiter가 슬라이딩 윈도우로 호출을 자체 억제. 한도 초과 시 최대 2초 대기, 그래도 못 뚫으면 KisRateLimitedException → KisExceptionHandler가 HTTP 429 + Retry-After 헤더로 변환 → 프론트 Toast가 자동 감지해 "X초 후 재시도" 배너 표시.

3.6 보안/환경변수 설계

키는 절대 파일 커밋 금지. application.yml은 ${HANTOO_API_KEY:} 형태로만 참조하고, 실제 값은 ~/.bashrc의 export로 쉘 환경변수에 주입. .gitignore에 .env 패턴 추가 완료.

4. 실제 투자에서의 활용 방안

4.1 전략 검증 파이프라인

  1. 가설 설정: "KOSPI 대형주 + 미국 S&P500 ETF 60:40, 분기 리밸런싱" 같은 전략을 문장으로 정의
  2. 포트폴리오 백테스트 탭에서 직접 입력 → 2010~현재 수익률, Sharpe, MDD, CAGR 확인
  3. 벤치마크 켜고 SPY/KODEX200/예금 대비 초과수익 확인
  4. 위기 시나리오 탭에서 GFC·COVID·금리인상 시 손실률 테스트
  5. Sharpe > 0.7 & MDD < -30% 견딜 수 있으면 실행 단계

4.2 실행 단계

  1. 자동매매 탭에서 대상 종목 감시 등록 → 실시간 가격 모니터링
  2. 리밸런싱 시점이 오면 수량 계산 후 지정가 주문
  3. 예상 비용 프리뷰로 수수료·세금 확인 (국내 매도 0.18%, 미국 0.25%)
  4. 주문 후 "미체결 주문" 패널에서 체결 여부 확인, 미체결은 한투 MTS에서 취소

4.3 투자자 동향 활용

월간·분기 리뷰 시 "투자자별 매매동향" 탭에서 보유 종목의 외국인·기관 동향 확인. BEARISH 신호(지속 순매도인데 주가 버티는 중)가 나오면 비중 축소 검토. BULLISH 신호(매집인데 주가 횡보) 나오면 분할매수 고려.

4.4 세금·수수료 최적화

주문 프리뷰가 매도 거래세를 실시간 보여주므로, 단타 빈도를 낮추는 것이 눈에 보이는 비용 절감. 또한 해외주식 양도세(22%, 연 250만원 공제)는 연간 실현손실과 상계 가능 → 연말에 손절 후 재매수하는 세금 최적화 전략(tax-loss harvesting)을 툴에서 계획 가능.

4.5 일일 운용 루틴 예시

시간작업도구
장전 (08:40)전일 체결·미체결 확인, 오늘 주문 계획 수립감사 로그, 잔고
장중 (09:00~15:30)감시종목 실시간 가격 체크, 조건 충족 시 수동 주문자동매매 탭
장마감 후투자자 동향 업데이트, 주 1회 백테스트 재검증투자자별 매매동향, 포트폴리오 백테스트
월말리밸런싱 계산 → 실행, 위기 시나리오 재실행포트폴리오, 위기 시나리오

5. 퀀트 툴로의 발전 로드맵

현재는 "백테스트 + 수동 실행"에 머물러 있음. 본격적인 퀀트 플랫폼으로 확장하려면 3단계로 접근 권장.

Phase 1 · 전략 엔진 도입 (1~2개월)

(a) 전략 추상화

현재 백테스트는 "비중 × 리밸런싱"만 지원. 다음을 추가:

  • Signal: 가격·지표로부터 "매수/매도/관망" 신호 생성 (이동평균 교차, RSI, 모멘텀 등)
  • Rule: Signal 조합 + 포지션 사이징 (Kelly, 변동성 타깃 등)
  • Execution: Rule → 실제 주문(수동 승인 or 자동)
interface Strategy {
  Signal evaluate(MarketContext ctx);
}
class MovingAverageCross implements Strategy { ... }
class RSIMeanReversion implements Strategy { ... }

(b) 팩터 라이브러리

밸류(PBR·PER), 모멘텀(3개월 수익률), 퀄리티(ROE), 사이즈(시가총액) 팩터를 KIS의 종목 재무 API(국내주식 재무비율, 국내주식 수익성비율)로 수집 → 스코어링.

(c) 벡터화 백테스트 엔진

현재는 Java 루프 기반이라 다종목·다기간 시 느림. Python으로 별도 백테스트 워커를 두고 (pandas, numpy) 일봉 데이터를 SQLite/Parquet에 캐시 → REST로 Java 서버와 통신.

Phase 2 · 실시간 자동화 (2~3개월)

(a) KIS WebSocket 직접 연동

현재는 30초 폴링. 진짜 실시간 체결가는 /oauth2/Approval로 approval_key 발급 후 ws://... WebSocket 연결. 41개 종목까지 동시 구독 가능. 국내주식 실시간체결가 등을 수신해 1틱 단위 반응.

(b) 조건부 주문 엔진

현재는 "사용자가 매수 버튼 누름". 확장:

  • "삼성전자 70,000원 이하면 10주 매수" — 가격 트리거
  • "RSI < 30 도달하면 분할매수 1차" — 지표 트리거
  • "동시 5종목 체결 조건 만족 시 바스켓 주문" — 포트폴리오 트리거

백엔드에 ConditionalOrderService(Redis·DB 저장) + OrderTriggerScheduler(매분 체크) 추가.

(c) 실거래 계정 분리·운영

전략별로 KIS 서브계좌 분리 → 성과·리스크 격리. 앱키도 전략별로 별도 발급해 Rate Limit 병목 회피(키 1개당 18/sec 독립).

Phase 3 · 본격 퀀트 플랫폼 (3~6개월)

(a) 데이터 레이크

현재는 요청마다 KIS API 호출. 확장:

  • 일봉/재무/지표를 TimescaleDB 또는 Parquet에 적재
  • 야간 배치로 증분 수집 (rate limit 위반 없이 분산)
  • 백테스트는 DB만 쿼리 → 10~100배 빠름

(b) 리스크 관리

  • VaR·CVaR (파라메트릭/몬테카를로)
  • 팩터 노출도 분석 (섹터·스타일 기여도)
  • Kelly 기준 최대 포지션 비중 자동 산정
  • 긴급 중단 장치(Kill Switch): 일일 손실률 초과 시 신규 주문 차단

(c) 전략 대시보드

현재는 수동 확인. 확장:

  • 라이브 P&L (전략별 실시간 손익)
  • 슬리피지 분석 (예상가 vs 체결가 차이)
  • 트랜잭션 비용 분석 (TCA)
  • 알림 (Slack/Discord 웹훅으로 체결·리스크 이벤트 푸시)

(d) 바스켓·페어 트레이딩

KR-US 종목 간 상관 분석 → 페어 트레이딩(코스피200 ↔ S&P500 ETF 스프레드). 현재 구조에서 자연스럽게 확장 가능한 영역.

(e) 머신러닝·대체 데이터

기본기 다진 후 선택적으로:

  • LSTM/Transformer 기반 단기 방향성 예측 (Feature: OHLCV + 투자자 동향 + 뉴스 감성)
  • KIS 해외뉴스종합, 종합 시황 공시 엔드포인트 + 공개 뉴스 API → NLP 감성 분석
